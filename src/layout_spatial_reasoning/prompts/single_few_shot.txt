Output requirements:
- Return only valid JSON.
- The JSON must contain only a top-level "sections" field.
- Do not use alternative field names such as "title", "items", "fields", "columns", or "children".
- Each row must be an object with "row_id" and "controls"; never make a row a raw array.
- Use the exact input control ids.

Rules:
- Use a twelve-column grid.
- Use exactly these output keys: sections, section_id, section_name, rows, row_id, controls, id, colStart, colSpan.
- Every input control id must appear exactly once.
- Do not invent control ids.
- colStart and colSpan must be integers in [1, 12].
- A control may not extend beyond column 12.
- Controls in the same row must not overlap.
- Group semantically related controls into coherent sections.
- Put semantically related short controls in the same row when possible.
- Preserve a natural form-completion reading order.
- Use full-width rows for long_text and file controls unless another layout is clearly better.
